# OMG - Open Modular Gardening – Session Log

- Date: 2026-03-02
- Duration: 02:00 (local)
- Related user stories: US19, US20, US21

## Context
This session introduced a full authentication and account lifecycle flow for the OMG backend, wired into the existing DDD architecture. The focus was on ASP.NET Core Identity, JWT access/refresh tokens, email verification via RabbitMQ, and securing garden/plant/telemetrics endpoints per user.

## What was done
- Added an `OMG.Auth.Infrastructure` project with `AuthDbContext`, `ApplicationUser`, `RefreshToken`, and EF Core migrations for the new `auth` schema.
- Configured ASP.NET Core Identity, JWT bearer authentication, and login rate limiting in `Program.cs`, plus central `Jwt` configuration in `appsettings*.json`.
- Implemented `AuthEndpoints` (register, login, refresh, logout, account deletion) and a `TokenService` that issues short-lived JWTs and rotating refresh tokens persisted in the database.
- Introduced `SendRegistrationEmail` integration contract and a `RegistrationEmailConsumer` that mocks email sending and auto-verifies accounts via RabbitMQ (demo-only behavior).
- Secured management and telemetrics endpoints to derive `UserId` from JWT claims, enforcing that users can only manage gardens and plants they own.
- Seeded demo users `bram@inthepocket.com` and `jonas@inthepocket.com` with roles and verified status via `AuthDbContextSeeder`.
- Added integration tests for auth flows and updated existing API tests to exercise secured endpoints using real JWTs instead of passing user IDs directly.
