## Context
Adjusted the management API so garden and plant updates support optional fields and only apply changes when specific properties are provided. This session focused on aligning the HTTP contracts and endpoint logic with the desired partial-update behavior.

## What was done
- Updated `UpdateGardenRequest` and `UpdatePlantRequest` DTOs to make all updatable properties nullable.
- Changed `UpdateGarden` endpoint logic to trim the name and only call domain methods when the corresponding nullable properties are supplied and differ from current values.
- Changed `UpdatePlant` endpoint logic to treat species, type, surface area, ideal humidity, and plantation date as optional, reclassifying or adjusting only when non-null values are provided and different.
- Ensured plant type parsing/validation only runs when a new type is passed, preserving previous behavior for invalid types.
- Created and pushed commit `5bc4509` on branch `main` with message `feat(management): support partial garden and plant updates`.
