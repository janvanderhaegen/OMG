## Context

Session added a GET-only reporting endpoint that produces a detailed garden/irrigation report using raw SQL on the existing ManagementDbContext and TelemetryDbContext. To support “plants added/deleted since date,” the plants table gained CreatedAt and DeletedAt with soft-delete behavior.

## What was done

- **GET /api/v1/reports/garden-report**: Single GET endpoint (Reports/ReportingEndpoints.cs) with optional query params: gardenId, from, to, lastMinutes, sinceDate. Returns watered count, unwatered count, watering frequency per plant in the period, and plants added/deleted since a date. All reads via Database.SqlQueryRaw on both contexts; scope restricted to current user’s gardens.
- **Plant schema**: PlantEntity and ManagementDbContext extended with CreatedAt (required) and DeletedAt (nullable); global query filter on plants so DeletedAt == null. GardenRepository sets CreatedAt on new plants and soft-deletes by setting DeletedAt instead of removing the row.
- **Migration AddPlantCreatedAndDeletedAt**: Adds CreatedAt (NOT NULL, defaultValueSql: now()) and DeletedAt to gm.plants.
- **DTOs**: ReportDtos.cs with WateringFrequencyItem(PlantId, Count) and GardenReportResponse (counts, frequency list, since date, period bounds). In-memory DB returns 200 with zeros.
- **Tests**: ReportEndpointTests (unauthorized, authenticated shape, no gardens, query params). PlantManagementEndpointTests plant entities now set CreatedAt where applicable.
