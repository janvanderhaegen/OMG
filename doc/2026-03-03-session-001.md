## Context

Session implemented the final rules for plant watering and telemetrics: domain repository for the plant hydration aggregate, telemetry endpoint reworked to use domain methods (RegisterCurrentHumidity, StartWatering, StopWatering), removal of ApplyMinuteTick, and isWatering in the irrigation simulation worker derived from WateringSession (started &lt;2 min ago, not ended). RabbitMQ messages for WateringStarted and WateringCompleted were added and published via the integration event publisher.

## What was done

- **IPlantHydrationStateRepository** added in `OMG.Telemetrics.Domain/Hydration/`. Interface defines GetByPlantIdAsync, GetByMeterIdAsync, ListWithIrrigationLineAsync (State, MeterId, GardenId), AddAsync, SaveAsync. Implementation remains in Infrastructure (`PlantTelemetryRepository`).
- **PlantHydrationState**: added `RegisterCurrentHumidity(int)` (validates 0–100, sets CurrentHumidityLevel); added `StopWatering(DateTimeOffset)` (completes active session, applies humidity increase, raises WateringCompletedDomainEvent; no-op if no session). Removed `ApplyMinuteTick`.
- **Repository**: GetByMeterIdAsync/GetByPlantIdAsync/ListWithIrrigationLineAsync now load active `WateringSessionEntity` and map to domain `WateringSession`; SaveAsync/AddAsync sync sessions (insert when ActiveSession set, mark ended when cleared).
- **TelemetryEndpoints**: ProcessReadingsForGardenAsync uses repository and domain: get/create aggregate, RegisterCurrentHumidity(reading.CurrentHumidity), StartWatering or StopWatering per reading.IsWatering, Save/Add, then publish collected domain events via ITelemetryIntegrationEventPublisher.
- **Messaging**: `WateringStarted` and `WateringCompleted` contracts in TelemetryEvents.cs; TelemetryIntegrationEventPublisher publishes them from WateringStartedDomainEvent and WateringCompletedDomainEvent.
- **IrrigationSimulationWorker**: Uses IPlantHydrationStateRepository.ListWithIrrigationLineAsync; computes isWatering via IsWateringFromSession (ActiveSession non-null, started &lt;2 min ago, utcNow &lt; EndsAt); builds TelemetryReadingRequest(meterId, next, isWatering). No longer mutates entities or SaveChanges; persistence happens in endpoint.
- **Tests**: Removed four ApplyMinuteTick tests; added tests for RegisterCurrentHumidity (valid + out-of-range) and StopWatering (completes session + event, no-op when no session).
- **DI**: Program.cs registers IPlantHydrationStateRepository and ITelemetryIntegrationEventPublisher.
