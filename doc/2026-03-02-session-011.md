## Context
Implemented a proper email verification flow for user registration, including verification codes, a confirmation endpoint, and a resend mechanism with rate limiting. Also aligned the auth model with ASP.NET Identity by consolidating on the built-in email confirmation flag.

## What was done
- Introduced a verification-code based email confirmation flow with `GET /api/v1/auth/verify-email` that validates codes and marks emails as confirmed.
- Added a `POST /api/v1/auth/resend-verification-email` endpoint that regenerates and resends verification codes while enforcing a per-user once-per-minute limit.
- Extended `ApplicationUser` and auth migrations to track verification code send times and removed the redundant `IsEmailVerified` flag in favor of `EmailConfirmed`.
- Updated registration to return the verification code in the response and to initialize the new tracking fields.
- Adjusted the mock `RegistrationEmailConsumer` to only log outgoing emails instead of auto-verifying users.
- Updated auth endpoint tests and helpers to cover the new verification and resend behavior and to rely solely on `EmailConfirmed`.
- Committed and pushed these changes on branch `main` as commit `3a49231b7c93f4911839382c0f2456917689553b`.
